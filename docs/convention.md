# 🤝 베리타스 협업 컨벤션

우리 동아리의 원활한 기록 관리와 생기부 작성을 위한 공통 규칙입니다.

## 1. 커밋 메시지 규칙 (Commit Message)
터미널에서 `git commit -m` 뒤에 쓸 메시지 형식입니다.
- **형식:** `[학년/이름] 작업내용`
- **예시:** `git commit -m "[1학년/홍길동] 2주차 판다스 실습 완료"`

## 2. PR 제목 규칙 (Pull Request)
멘토에게 제출할 때 쓰는 제목입니다.
- **형식:** `[제출] 학년_이름_주차/주제`
- **예시:** `[제출] 1학년_홍길동_02_Pandas기초`

## 3. 파일 및 폴더 관리
- 모든 개인 과제는 반드시 `submissions/[학년]_year/[이름]/` 폴더 안에 넣습니다.
- 파일명은 멘토가 배포한 이름을 유지하되, 필요시 뒤에 날짜를 붙입니다. (예: `day1_analysis_260313.ipynb`)

## 4. 코드 작성 에티켓
- **주석 달기:** 코드가 왜 이렇게 작성되었는지 간단한 설명을 `#` 뒤에 적어주세요.
- **질문하기:** 막히는 부분은 `.github` 폴더의 이슈 템플릿을 활용해 적극적으로 질문하세요.
